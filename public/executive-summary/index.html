AI Agent Capability Maturity Model

A structured framework for adopting AI coding agents across the software development lifecycle -- safely, measurably, and at scale.

What is this?

Purpose

A maturity model that helps organizations adopt AI agents (Copilot, Claude Code, Cursor, Codex, and similar tools) deliberately rather than by uncontrolled adoption or blanket prohibition. It provides a structured path from initial experimentation to optimized, governed usage.

Scope

Covers the full software development lifecycle: local development, code review, CI/CD, infrastructure as code, operations, security, knowledge management, governance, and measurement. Platform-agnostic -- works with any cloud, SCM, or CI/CD platform.

Who should use it

Engineering leadership, platform teams, security teams, and governance bodies responsible for AI agent adoption decisions. Useful for organizations of any size that want to move beyond ad hoc agent usage.

What you get

A current-state assessment score (0-5) across 9 domains, identification of critical gaps and risks, and a tailored 90-day improvement plan with specific actions and milestones.

Maturity Levels

5
Adaptive
The org continuously tunes agent behavior, context, and guardrails using feedback loops and measured outcomes. Proactive capabilities and industry leadership.
4
Optimized
Agents are integrated into CI/CD, IaC, and ops workflows with automated governance. Policy-as-code enforcement. Full KPI coverage and ROI analysis.
3
Managed
Agent usage is instrumented. Metrics are collected and reviewed. Workflows are standardized with templates and reusable components.
2
Defined
Policies exist. Approved tools are listed. Basic guardrails are in place. This is the minimum safe level for expanding agent usage.
1
Exploratory
Individuals experiment with AI agents. No org-level policy or tooling. Usage is ungoverned and risk is unmanaged.
0
Unaware
No intentional use of AI agents. Shadow usage may exist without organizational awareness.

9 Capability Domains

Domain 1

Dev Workflow

How agents are used during local development: IDE integration, dev environments, CLI tooling, and configuration.

Domain 2

PR and Code Review

How agent-generated code is disclosed, reviewed, labeled, and merged. Review rigor and quality tracking.

Domain 3

CI/CD

How agents participate in build, test, and deploy pipelines. Isolation, auditability, and pipeline standards.

Domain 4

Infrastructure as Code

How agents generate, modify, and validate IaC. Module registries, policy-as-code, and drift detection.

Domain 5

Ops and Incident Response

How agents assist during incidents, runbooks, and operational tasks. Safety rules and remediation boundaries.

Domain 6

Security and Compliance

How agent usage meets security policies, data classification, and regulatory requirements. DLP and audit.

Domain 7

Knowledge Management

How agents consume and contribute to documentation, ADRs, and organizational context. RAG and knowledge bases.

Domain 8

Governance

How the org manages agent access, approvals, audit, cost tracking, and exceptions. Policy enforcement.

Domain 9

Evaluation and Measurement

How the org measures agent value, code quality, ROI, and risk. KPI definitions and decision-driven metrics.

Key Principles

  • A domain earns a level only if all required practices for that level are in place. No skipping levels.
  • Security and Compliance must reach Level 2 before expanding agent usage. No exceptions.
  • Governance must reach Level 2 for other domain scores to be reliable.
  • Any domain 2+ levels below the overall average is a critical gap requiring immediate attention.
  • Regulated environments (FedRAMP, HIPAA, PCI-DSS, GDPR) must be assessed separately if tooling differs.
  • The model is platform-agnostic. Adapt it to your cloud, SCM, CI/CD, and tooling stack.
  • 10 categories of high-risk changes always require human review regardless of maturity level.
  • Measurement without action is waste. Every KPI should have an owner and a decision threshold.

Getting Started

  1. Assemble a cross-functional assessment team: engineering, security, platform, and compliance representatives.
  2. Complete the interactive assessment -- score each of the 9 domains from Level 0 to Level 5 using the evidence checklists and interview questions provided.
  3. Review the results dashboard: overall score, radar chart, critical gaps, and security/governance warnings.
  4. Use the generated 90-day improvement plan to prioritize your first actions, starting with governance and security foundations.
  5. After the 90-day cycle, reassess to measure progress and set targets for the next cycle.
